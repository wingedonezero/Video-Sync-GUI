SESSION HANDOFF — GPU Dense Correlation Overhaul
=================================================

WHAT WE DID
-----------

1. PORTED ALL CORRELATION METHODS TO GPU (torch/torchaudio)
   - SCC, GCC-PHAT, GCC-SCOT, GCC-Whiten, Onset Detection, Spectrogram
   - DTW removed entirely (deleted from registry, method file still exists as orphan)
   - numpy→torch conversion happens inside each method's find_delay()
   - torchaudio transforms cached in gpu_backend._transform_cache
   - Lazy imports of torch/torchaudio inside find_delay() methods
   - GPU cleanup (cleanup_gpu()) called after each job

2. REPLACED 30-CHUNK WITH DENSE SLIDING WINDOW
   - Old: 30 evenly spaced chunks → correlate each → pick delay
   - New: sliding window (10s window, 2s hop) across entire file → 600+ results
   - dense.py is the ONLY correlation mode now (legacy chunk path deleted)
   - Settings: dense_window_s=10.0, dense_hop_s=2.0, dense_silence_threshold_db=-60.0
   - Silence windows auto-skipped

3. REWORKED DELAY SELECTION FOR DENSE DATA
   - Modes: Mode (Most Common), Mode (Clustered), First Stable, Average
   - First Stable: finds dominant delay in first 25% of windows
   - Mode (Clustered): clusters within ±1ms tolerance
   - All modes work on full dense result set (not 30 chunks)

4. FIXED GCC-PHAT FALSE PEAKS (bandpass filter)
   - Problem: PHAT/SCOT/Whiten produced false peaks at 0ms and ±10000ms
   - Root cause: out-of-band frequency bins (<300Hz, >6kHz) have ambiguous phase;
     PHAT normalizes all bins to unit magnitude, giving these noise bins equal weight
   - SCC doesn't have this because it preserves energy weighting
   - Fix: bandpass_mask() in gpu_correlation.py — zeros out bins outside 300Hz-6kHz
     in frequency domain BEFORE PHAT weighting, then re-zeros after normalization
   - Applied to all 3 phase-only methods (gcc_phat.py, gcc_scot.py, gcc_whiten.py)
   - SCC untouched — doesn't need it
   - Tested: 16 false peaks → 0 on stepping file, real delays unchanged
   - This is what NGCC-PHAT's SincNet layer does — learns a bandpass filter

5. PSR CONFIDENCE (replaced broken confidence scoring)
   - Old: normalize_peak_confidence_torch and scot_confidence — gave 100% to everything
   - New: psr_confidence() — Peak-to-Sidelobe Ratio from radar/sonar literature
   - PSR = (peak - mean_sidelobes) / std_sidelobes
   - Thresholds: PSR ≤ 10 → 0%, PSR ≥ 20 → 100% (noise floor ~8-10 at 48kHz)
   - Note: PSR alone doesn't fix PHAT false peaks (they have PSR ~950 because
     phase cancellation produces sharp peaks, not noise). Bandpass is the real fix.

6. SYNC STABILITY DEFAULTS FOR DENSE DATA
   - Old: variance_threshold=0.0, outlier_mode="any" → flagged 0.0005ms as unstable
   - New: variance_threshold=1.0, outlier_mode="threshold"
   - Dense produces tiny natural variance across 600+ windows; old settings were
     calibrated for 30 chunks

FAILED APPROACHES (don't repeat these)
---------------------------------------
- max_lag_samples restriction on extract_peak() — clips real delays >10s
- Valid lag region confidence scoring — PHAT correlation is uniform in valid region
- PSR alone for false peak rejection — false peaks have PSR ~950 on real audio
